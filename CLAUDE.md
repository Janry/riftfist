# Проєкт: Beat 'em up (робоча назва)

Контекст-передача з дизайн-сесії. Це постійні інструкції для Claude Code.
Тримати цей файл коротким; деталі — в `ROADMAP.md` і в прототипі.

## Що це
2D side-scrolling beat 'em up у стилі Double Dragon / Battletoads.
Сюжет — подорожі по вимірах (оммаж до «Рік і Морті»), кожен вимір = рівень.
Ціль платформи: **iPhone (iOS)**, керування — тач.

## Жорсткі обмеження (не порушувати без явного рішення)
- **IP / ліцензія:** НЕ використовувати реальні імена/персонажів/арт «Рік і Морті».
  Сетинг — оригінальний/пародійний «у дусі». Все IP-залежне (імена, спрайти,
  репліки, теми вимірів) тримати у свапабельному data-шарі (`THEME`), окремо
  від рушія, щоб «версію-страховку» можна було зробити підміною даних.
- **Кооп vs тач:** локальний кооп на одному екрані айфона не лізе (нема місця
  під 2 набори контролів). Рішення відкладене: або соло на телефоні, або
  кооп на десктоп/онлайн. Не закладати локальний 2P на одному пристрої.

## Поточний статус
Механіка підтверджена на HTML-прототипі (`beatemup-prototype.html` — читати
як дизайн-специфікацію). Прототип одноразовий; не розвивати його далі —
він лише еталон «відчуття бою».

## Найближче завдання
Порт ядра прототипу в **Godot 4 (GDScript)** зі збереженням game feel.
Перед кодом: переглянути `beatemup-prototype.html`, потім перенести в такому
порядку: рух у глибину → автомат станів гравця + хітбокси → hitstop/тряска →
вороги (AI) → стрибок/повітряний удар → зброя → knockdown → бос.

## Підтверджена механіка (зберегти відчуття при порті)
Псевдо-3D: x = горизонталь, y = глибина (рух по y повільніший ×0.7),
сортування спрайтів по y. Влучання вимагає збігу і по x, і по глибині.

Ключові «ручки» відчуття (з прототипу — звіряти точні числа там):
- **hitstop**: 4–10 кадрів заморозки логіки на влучання (КРИТИЧНО для соковитості).
- **screenshake** + білий флеш цілі на кожен удар.
- **комбо-ланцюг** a1→a2→a3 через вікна скасування; 3-й удар важкий.
- **спец** коштує HP; **стрибок**: GRAV=0.8, JUMP_V=14; у високому стрибку
  ворожі удари не влучають (z>22) — стрибок і оборонний.
- **зброя** (data-шар `WEAPONS`): підбір на дотик, озброєний удар = мах,
  озброєний спец = кидок-снаряд; міцність у «зарядах», ламається.
- **knockdown**: важкі атаки (a3, спец, повітряний, кидок; прапор `down`)
  валять ворога; слабкі — лишають на ногах. Бос НЕ валиться (важкий).
- **бос**: кожна 3-тя хвиля, більше HP, довша дистанція, своя смужка.
